# 資料分析、清理與建模過程紀錄

## 1. 專案目標

本專案目標是建立一個小型資料處理 Pipeline，流程包含：

1. 使用 Python Selenium 爬取 Yahoo 奇摩娛樂即時新聞。
2. 擷取新聞標題、新聞連結、新聞來源與發布時間。
3. 進入每篇新聞頁面擷取新聞內文。
4. 使用 LLM 對新聞內容進行摘要、實體辨識與分類。
5. 將整理後資料輸出為 CSV / JSON。
6. 使用 Docker 封裝執行環境。

---

## 2. 資料來源

資料來源為 Yahoo 奇摩新聞娛樂即時頁面：

```text
https://tw.news.yahoo.com/entertainment/archive/
```

原始頁面包含娛樂即時新聞列表，每則新聞通常包含：

- 新聞標題
- 新聞連結
- 新聞來源
- 相對發布時間，例如「幾分鐘前」、「1 小時前」

---

## 3. 資料爬取流程

本專案使用 Selenium 開啟 Yahoo 娛樂即時新聞頁面，並透過瀏覽器 DOM 擷取新聞列表。

主要步驟：

1. 開啟 Yahoo 娛樂即時新聞頁面。
2. 等待頁面載入完成。
3. 嘗試關閉同意或隱私提示視窗。
4. 從頁面中擷取符合 Yahoo 新聞網址格式的文章連結。
5. 過濾重複連結。
6. 從每篇新聞卡片中取得新聞標題、來源與發布時間。
7. 進入每篇新聞連結，擷取文章正文內容。

---

## 4. 資料清理方式

資料清理包含以下處理：

### 4.1 新聞列表清理

- 使用新聞網址格式過濾非新聞連結。
- 移除重複 URL。
- 過濾沒有標題或沒有時間資訊的項目。
- 避免將側邊欄、熱門新聞或廣告區塊誤判為主要新聞。

### 4.2 發布時間處理

Yahoo 頁面上的時間為相對時間，例如：

```text
12 分鐘前
1 小時前
剛剛
昨天
```

程式會將相對時間轉換為推估發布時間，方便後續排序與輸出。

### 4.3 新聞內文清理

進入新聞頁面後，程式會擷取正文段落，並移除常見雜訊，例如：

- 延伸閱讀
- 相關新聞
- 看更多
- Yahoo APP 推廣文字
- 重複段落
- 過短或非正文內容

---

## 5. LLM 分析與標註

本專案將新聞標題與新聞內文輸入 LLM，並要求模型輸出 JSON 格式結果。

LLM 分析任務包含：

### 5.1 新聞摘要

將新聞內文整理成 80 字以內的繁體中文摘要。

### 5.2 NER 實體辨識

抽取新聞中出現的人名與團體名，例如：

```text
五月天
周杰倫
張惠妹
BLACKPINK
Energy
```

若未出現明確人名或團體名，則填入 `Null`。

### 5.3 演唱會分類

判斷新聞是否主要與演唱會相關。

若新聞內容涉及以下主題，則分類為「是」：

- 演唱會
- 開唱
- 巡演
- 售票
- 加場
- 嘉賓
- 場館
- 演出事故

否則分類為「否」。

---

## 6. 輸出資料欄位

清理與分析後的 CSV 包含以下欄位：

| 欄位名稱 | 說明 |
|---|---|
| 新聞標題 | Yahoo 新聞標題 |
| 新聞連結 | 原始新聞網址 |
| 新聞來源 | Yahoo 顯示的新聞來源 |
| 發布時間文字 | 原始相對時間文字 |
| 推估發布時間 | 轉換後的發布時間 |
| 新聞內文 | 擷取後的新聞正文 |
| 新聞內文摘要 | LLM 產生的摘要 |
| 實體(人名/團體) | LLM 擷取的人名或團體名，多個以分號分隔 |
| 是否為演唱會 | LLM 判斷是否與演唱會相關 |

---

## 7. Docker 封裝

本專案提供 Dockerfile，將 Python、Selenium、Chromium 與必要套件封裝在同一個執行環境中。

建置 Docker image：

```bash
docker build -t yahoo-entertainment-scraper .
```

執行 Docker container：

```bash
docker run --rm \
  -e OPENAI_API_KEY="your_api_key_here" \
  -v "$PWD/output:/app/output" \
  yahoo-entertainment-scraper
```

執行完成後，CSV / JSON 結果會輸出至 `output/` 資料夾。

---

## 8. 實作限制與觀察

實測時發現，Yahoo 娛樂即時頁面目前主要載入最新一批娛樂新聞，往下滑後不一定會繼續載入更舊的娛樂即時新聞，而可能轉為其他新聞區塊或推薦內容。

因此本專案以 Yahoo 娛樂即時頁面目前可取得的主列表新聞為主要資料來源，並完成新聞內文擷取、資料清理、LLM 摘要、NER 與演唱會分類。

若未來需要更完整的 12 小時資料，可進一步研究 Yahoo 背後是否有可用的 pagination API 或 JSON 資料接口。
